# §7.3.7 / Annexe D — SEI Messages

Spec ref : §7.3.7 (SEI RBSP syntax), Annexe D (SEI message semantics)

## Priorité : BASSE

Les SEI (Supplemental Enhancement Information) messages ne sont pas nécessaires pour le décodage. Ils fournissent des métadonnées optionnelles.

## Messages SEI utiles (à implémenter si besoin)

| payloadType | Nom | Utilité |
|-------------|-----|---------|
| 1 | pic_timing | Timing info pour l'output |
| 5 | user_data_unregistered | UUID + données custom |
| 129 | active_parameter_sets | Indique les parameter sets actifs |
| 132 | decoded_picture_hash | MD5/CRC/checksum du frame décodé |
| 137 | mastering_display_colour_volume | HDR metadata |
| 144 | content_light_level_info | HDR max content/frame light level |

## decoded_picture_hash (payloadType 132)

Particulièrement utile pour la validation :

```
decoded_picture_hash() {
    hash_type    u(8)   // 0=MD5, 1=CRC, 2=checksum
    for(cIdx = 0; cIdx < (ChromaArrayType == 0 ? 1 : 3); cIdx++) {
        if(hash_type == 0)
            picture_md5[cIdx][0..15]    b(8) x16
        else if(hash_type == 1)
            picture_crc[cIdx]            u(16)
        else
            picture_checksum[cIdx]       u(32)
    }
}
```

Cette SEI permet de vérifier le décodage indépendamment de l'oracle libde265 !
