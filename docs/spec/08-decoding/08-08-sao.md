# 8.7.3 — Sample Adaptive Offset (SAO)

Spec ref : 8.7.3.1 (general), 8.7.3.2 (edge offset), 8.7.3.3 (band offset)

## Vue d'ensemble

SAO est un filtre post-deblocking qui applique des offsets aux samples pour reduire les artefacts de quantification. Il opere par CTU, avec des parametres signales dans le bitstream.

### Modes SAO

| sao_type_idx | Mode | Description |
|-------------|------|-------------|
| 0 | Off | Pas de SAO pour ce CTU/composante |
| 1 | Band Offset | Offset par bande d'amplitude |
| 2 | Edge Offset | Offset base sur la comparaison avec les voisins |

## Parametres SAO par CTU (7.3.8.3)

```
sao(rx, ry) {
    // Pour chaque composante cIdx (Y, Cb, Cr) :
    if (cIdx == 0)
        sao_merge_left_flag    ae(v)
    if (cIdx == 0 && !sao_merge_left_flag)
        sao_merge_up_flag      ae(v)

    if (!merge) {
        sao_type_idx            ae(v)  // 0=off, 1=band, 2=edge
        if (sao_type_idx != 0) {
            for (i = 0; i < 4; i++)
                sao_offset_abs[i]    ae(v)
            if (sao_type_idx == 1) {
                // Band offset
                for (i = 0; i < 4; i++)
                    sao_offset_sign[i]   ae(v)
                sao_band_position        ae(v)
            } else {
                // Edge offset : signs are fixed by the category
                sao_eo_class             ae(v)  // 0=H, 1=V, 2=diag135, 3=diag45
            }
        }
    }
}
```

## 8.7.3.2 — Edge Offset

### EO Classes (directions)

```
Class 0 (Horizontal)  : p-1,0  p0,0  p1,0
Class 1 (Vertical)    : p0,-1  p0,0  p0,1
Class 2 (Diagonal 135): p-1,-1 p0,0  p1,1
Class 3 (Diagonal 45) : p1,-1  p0,0  p-1,1
```

### Categories et offsets

```cpp
// Pour chaque sample p(x,y) du CTU :
int a = ref_sample_1;  // sample voisin dans la direction EO
int b = ref_sample_2;  // sample voisin oppose
int c = current_sample;

// Categorisation
int edgeIdx;
if (c < a && c < b) edgeIdx = 0;      // Valley
else if (c < a || c < b) edgeIdx = 1;  // Concave (partial valley)
else if (c == a && c == b) edgeIdx = 2; // Flat (no offset)
else if (c > a || c > b) edgeIdx = 3;  // Convex (partial peak)
else edgeIdx = 4;                       // Peak

// Application de l'offset
// Les offsets pour EO sont contraints :
// Categorie 0 (valley) : offset > 0
// Categorie 1 (concave) : offset > 0
// Categorie 2 (flat) : offset = 0 (toujours)
// Categorie 3 (convex) : offset < 0
// Categorie 4 (peak) : offset < 0
const int sao_offset_val[5] = {
    +sao_offset[0],   // valley
    +sao_offset[1],   // concave
    0,                 // flat
    -sao_offset[2],   // convex
    -sao_offset[3]    // peak
};

output_sample = Clip1(current_sample + sao_offset_val[edgeIdx]);
```

## 8.7.3.3 — Band Offset

Divise la plage d'amplitude [0, 2^BitDepth-1] en 32 bandes. 4 bandes consecutives recoivent des offsets :

```cpp
void apply_band_offset(int16_t* samples, int stride, int width, int height,
                        const int offsets[4], int band_position, int bit_depth) {
    int max_val = (1 << bit_depth) - 1;

    for (int y = 0; y < height; y++) {
        for (int x = 0; x < width; x++) {
            int sample = samples[y*stride + x];
            int band = sample >> (bit_depth - 5);  // 32 bandes

            int band_idx = band - band_position;
            if (band_idx >= 0 && band_idx < 4) {
                samples[y*stride + x] = Clip3(0, max_val, sample + offsets[band_idx]);
            }
        }
    }
}
```

## SAO Merge

Les parametres SAO peuvent etre copies du CTU de gauche (`sao_merge_left`) ou d'au-dessus (`sao_merge_up`), evitant de re-signaler les memes parametres.

## Ordre d'application

```
1. Deblocking filter sur toute la picture
2. SAO sur toute la picture (utilise les samples post-deblocking)
```

SAO opere sur les samples **apres** deblocking.

## Pieges connus

1. **Bords de CTU** : Les comparaisons EO aux bords de CTU utilisent les samples du CTU voisin (apres deblocking, pas apres SAO du voisin)
2. **Cross-slice/tile** : Controle par `slice_loop_filter_across_slices_enabled_flag`
3. **Chroma sub-sampling** : La taille du CTU chroma est CtbSizeY/SubWidthC x CtbSizeY/SubHeightC
4. **Merge left/up** : Copie les 3 composantes en meme temps (luma + 2 chroma)
